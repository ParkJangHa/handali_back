## 1. 프로젝트 소개

### 📱 서비스 개요
**한달이**는 사용자의 일상 활동 데이터를 기반으로 가상 캐릭터가 성장하는 습관 관리 애플리케이션입니다.

단순한 체크리스트가 아닌 **게임화(Gamification)** 요소를 통해 습관 형성에 재미와 동기부여를 제공합니다.

<br/>

### 🎯 개발 배경

**기존 습관 앱의 문제점**
- 단순한 체크리스트 형태로 지속적인 동기 부여 부족
- 시각적 피드백 미흡
- MZ세대가 선호하는 감성적 요소 부재

**우리의 해결책**
- ✅ 캐릭터 성장으로 감정적 몰입 유도
- ✅ 216가지 외형 변화로 시각적 보상 제공
- ✅ 아파트 시스템으로 장기적 성취감 제공
- ✅ 코인 경제와 꾸미기 요소로 지속 사용 유도

<br/>

### 💡 핵심 컨셉
```
캐릭터(한달이) 생성 → 매일 습관 기록 → '한달이' 성장 → 외형 변화 → (한 달 경과후, 매달 1일)직업 획득 → 아파트 입주 → 새로운 '한달이' 시작
```

<br/>

## 2. 핵심 기능

### 🏃 습관 등록 및 기록

**3가지 카테고리로 구분된 습관 관리**

| 카테고리 | 예시 습관 |
|---------|----------|
| 🏃 활동 | 운동, 걷기, 요가, 스트레칭 |
| 📚 지능 | 독서, 강의 듣기, 코딩, 외국어 |
| 🎨 예술 | 그림, 악기 연주, 글쓰기, 사진 |

**기록 시 입력 항목:**
- 수행한 습관 (카테고리 - 세부 습관)
- 수행 시간
- 성취 만족도 (0~100점)

<br/>

### 🌱 스탯 계산 및 성장 시스템

습관 기록은 **복합 배율 공식**으로 스탯에 반영됩니다.
```
총 스탯 = (기록 횟수 × 8.5) + (만족도 × 1.5) + (시간 증가 × 1.5) + (이전 달 스탯 × 0.105)
```

**레벨 달성 기준 (하루 2시간, 만족도 50 기준)**

| 주차 | 일차 | 필요 스탯 | 레벨 |
|-----|------|----------|------|
| 1주차 | 2일 | 11.50 | Lv.1 ⭐ |
| 2주차 | 5일 | 28.75 | Lv.2 ⭐⭐ |
| 3주차 | 8일 | 46.00 | Lv.3 ⭐⭐⭐ |
| 4주차 | 13일 | 74.75 | Lv.4 ⭐⭐⭐⭐ |
| 5주차 | 18일 | 103.50 | Lv.5 ⭐⭐⭐⭐⭐ |

<br/>

### 🎨 216가지 캐릭터 외형 변화

스탯 레벨에 따라 **캐릭터 외형**이 자동으로 진화합니다.
```
활동 (6단계) × 지능 (6단계) × 예술 (6단계) = 216가지 조합
```

| 스탯 | 외형 변화 |
|------|----------|
| 🏃 **활동** | 땀방울 표시 → 작은 아령 → 중간 아령 → 큰 아령 → 바벨 |
| 📚 **지능** | 새싹 → 잎사귀 → 꽃봉오리 → 활짝 핀 꽃, 안경 착용 |
| 🎨 **예술** | 심플한 무늬 → 화려한 문양 → 알록달록한 색상 |

**예시:**
- 활동 Lv.3 + 지능 Lv.2 + 예술 Lv.1 = 중간 아령 + 잎사귀 + 심플 무늬

<br/>

### 💼 직업 시스템

**한 달 후 (매월 1일) 직업 배정**

1. **스탯 분석**: 활동, 지능, 예술 중 가장 높은 스탯 확인
2. **직업 배정**: 해당 스탯 기준 ±2단계 범위에서 랜덤 배정
3. **주급 지급**: 직업별로 차등 지급 (코인)

**직업 예시:**

| 스탯 | 직업 예시 | 주급 (코인) |
|------|----------|------------|
| 활동 | 운동선수, 요가 강사, 피트니스 트레이너 | 50~200 |
| 지능 | 개발자, 작가, 교수 | 100~300 |
| 예술 | 화가, 뮤지션, 디자이너 | 80~250 |

<br/>

### 🏢 아파트 입주 시스템

**직업 획득 후 자동 입주**

- 📅 **층 구조**: 연도별 12층 (1년 = 1개 동 = 12개월 = 12층)
- 🏠 **입주 위치**: 한달이 생성 월 = 층 수 (예: 5월 생성 → 5층)
- 💰 **주급 지급**: 입주한 한달이가 매주 코인 벌어옴
- 🎓 **독립 시스템**: 1년 후 완전 독립 (더 이상 주급 없음)

**아파트 기능:**
- 과거 한달이들을 층별로 보관
- 클릭 시 별명, 직업, 스탯, 주급 확인 가능
- 적용했던 가구 그대로 유지

<br/>

## 6. DB 모델링

<img width="1141" height="700" alt="스크린샷 2025-11-17 오후 12 40 13" src="https://github.com/user-attachments/assets/b4693bcc-f7cb-46f6-a62a-1506e643c142" />

### 📊 엔티티 구조 (15개)

**핵심 도메인 (6개)**
- `User`: 사용자
- `Handali`: 한달이 캐릭터
- `Habit`: 습관 마스터
- `Record`: 습관 기록
- `Job`: 직업
- `Apart`: 아파트

**관계 테이블 (5개)**
- `UserHabit`: 사용자-습관 연결
- `UserHandbook`: 사용자-도감 수집
- `HandaliStat`: 한달이-스탯 연결
- `UserStore`: 상점 구매 내역
- `UserItem`: 아이템 소유 및 적용

**참조 데이터 (4개)**
- `Handbook`: 도감 (216종)
- `Stat`: 스탯 (활동/지능/예술)
- `Store`: 상점 아이템
- `JobStat`: 직업-스탯 요구사항

### 🔑 주요 설계 원칙

- **N+1 문제 예방**: 모든 연관관계에 `FetchType.LAZY` 적용
- **참조 무결성**: `@ForeignKey`로 CASCADE 옵션 명시
- **Soft Delete**: User 엔티티에 `is_deleted` 컬럼
- **중복 방지**: `@UniqueConstraint`로 제약 조건 설정

## 8. 성능 최적화

### 🚀 JWT 토큰 구조 개선

**Before**
```
매 API 요청마다 Redis 블랙리스트 조회
→ Redis 부하 증가, 응답 지연
```

**After**
```
Access Token (15분) + Refresh Token (7일) 분리
→ Redis 조회 최소화
```

**결과**
- ✅ API 응답 **6.8% 개선**
- ✅ Redis 메모리 **80% 절감**

<br/>

### 🔥 JPA N+1 문제 해결

**Before**
```sql
-- 도감 조회 시
SELECT * FROM handbook;         -- 1개
SELECT * FROM user_handbook ...  -- 150개 (각 도감마다)
→ 총 151개 쿼리
```

**After**
```sql
-- Fetch Join 적용
SELECT h, uh FROM Handbook h 
LEFT JOIN FETCH h.userHandbooks uh
→ 총 3개 쿼리
```

**결과**
- ✅ 쿼리 **98% 감소** (151개 → 3개)
- ✅ API 응답 **66% 개선** (85ms → 29ms)

<br/>

### 📈 DB 인덱스 최적화

**추가 인덱스**
```sql
CREATE INDEX idx_handali_start_date ON handali(start_date);
```

**대상 쿼리**
```sql
-- 월별 한달이 조회
SELECT * FROM handali 
WHERE start_date >= '2025-01-01' 
  AND start_date < '2025-02-01';
```

**결과**
- ✅ Full Table Scan → Range Scan (EXPLAIN 검증)
- ✅ 월별 조회 **57% 개선** (4.0ms → 1.7ms)

<br/>

### 🧪 부하 테스트 검증 (k6)

**테스트 환경**
- 동시 사용자: 10명 → 50명 → 100명 (단계적 증가)
- 측정 시간: 3분

**결과 (도감 조회 API)**

| 지표 | Before (N+1) | After (해결) | 개선율 |
|------|-------------|-------------|--------|
| 평균 응답시간 | 60.99ms | 20.86ms | **66% ↓** |
| p95 응답시간 | 86.13ms | 34.63ms | **60% ↓** |
| TPS | 44.34/s | 46.18/s | 4% ↑ |
| 성공률 | 100% | 100% | - |
